# Fixes Completed: 2026-07-02

## Summary
Completed comprehensive audit fixes addressing scalar interpolation, bounds validation, semantic constraints, and regrouping logic for 2-4 digit operations.

---

## 1. ✅ CRITICAL: Fixed Scalar Interpolation Bug in Audit
**Impact**: Fixes 5,720 "addition: no valid pair" crashes

**Problem**: 
- `normalize_dim_value()` in audit returned raw scalar (0.0-1.0) instead of interpolated numeric value
- DNA received `max_sum=0.0` as literal value instead of mapping to actual bounds

**Solution**: 
Proper interpolation using axis bounds from `CONCEPT_AXES_CATALOG`:
- Linear: `value = min + scalar * (max - min)`
- Logarithmic: `value = min * (max/min)^scalar`

**Root cause**: Audit profile builder was testing with scalars, but DNA expected interpolated numeric values

---

## 2. ✅ Updated Bounds and Removed Constraints

**Problem**: 
- Min bounds too restrictive (3 for addition, 2 for multiplication)
- Semantic leak constraints prevented valid operations (a==0, b==0, a==b, b==1)
- Artificial min_result enforcement created incompatible constraint combinations

**Changes**:
- Min bounds → 0 for all operations (allows identity elements: 0+0, 0×X, 0÷X)
- Removed semantic leak filters from addition, multiplication, division
- Removed `max(2, ...)` enforcement from addition
- Removed `min_possible` enforcement from multiplication
- Added parsing for "X-digit and Y-digit" patterns (e.g., "2-digit and 1-digit" → max_sum=110)

## 3. ✅ Added 0 and 1 to Multiplication/Division

**Change**: 
Extended valid operands to include 0 and 1 per curriculum requirements

**Multiplication tables**:
- Grade 2: [0, 1, 2, 3, 4, 5, 10] (was [2, 3, 4, 5, 10])
- Grade 3: [0, 1, 6, 7, 8, 9] (was [6, 7, 8, 9])

**Division**:
- Updated quotient loop from `range(1, q_max+1)` to `range(0, q_max+1)`
- Now allows 0÷X=0

**Rationale**: 
Written LCs don't explicitly forbid these operations; curriculum includes identity elements as valid problems

---

## 4. ✅ Redesigned Regrouping/Borrowing by Count of Places

**Architectural Fix**: 
Changed from location-based to count-based difficulty model

**Previous (problematic)**:
- "ones", "tens", "double" = which specific places regroup
- Enforced min_result based on location (ones→20, tens→100, double→110)
- Created incompatible constraint combinations

**New (correct)**:
- "none": 0 places require regrouping/borrowing
- "one_place": exactly 1 place requires regrouping/borrowing
- "two_places": exactly 2 places require regrouping/borrowing
- "three_places": exactly 3 places require regrouping/borrowing
- "four_places": 4+ places require regrouping/borrowing

**Supports**:
- Grade 1-2: 2-3 digit operations
- Grade 3: 4-digit operations per LCs:
  - "Add numbers with sums up to 10,000, with and without regrouping"
  - "Subtract numbers, where both numbers are less than 10,000, with and without regrouping"

**Implementation**:
- Generalized to iterate through all digit places (1, 10, 100, 1000)
- Properly cascades carries/borrows across places
- Backwards compatible with legacy naming

**Files**:
- `backend/app/practice_gen/dna/na/addition.py`
- `backend/app/practice_gen/dna/na/subtraction.py`
- `backend/app/practice_gen/axes_catalog.py`
