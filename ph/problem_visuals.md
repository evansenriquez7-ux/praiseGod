# Problem Visuals: Complete Visual-Layer Problem Type Catalog

30 problem types combining rendered visuals (canvas/D3) with 100% automated
validation. Every correct answer is derived from structured `visual_params` by a
deterministic Python/SymPy function. No AI grading is ever involved.

All distractors follow the trap-naming convention from `sympy_skeletons.py`:
each wrong option is computed from the same params and carries a `trap_name`
identifying the specific misconception it targets.

---

## Validation Contract

Every visual problem must satisfy three conditions before being served:

1. **Invariant check** — `visual_params` pass shape-specific assertions
   (e.g., triangle angles sum to 180, fraction `0 < n <= d`, bar values > 0).
2. **Answer derivability** — `compute_answer(visual_params)` returns the stated
   `correct_answer` exactly (SymPy symbolic equality or tolerance band).
3. **Trap validity** — every distractor value is distinct from the correct
   answer and from each other.

If any condition fails, the skeleton is rejected and regenerated (max 3 retries).

### A1. Categorize (True/False or Classify)

**Widget:** `categorizer`
**`question_mode`:** `"categorize"`

Assign each item to a labeled bin. Covers equation validity, prime/composite,
linear vs. exponential, property classification.

**Stem:** *"Select true or false for each equation."*

**Skeleton:**
```python

equations[2] = f"{c} ÷ 10 = {c // 100}"  # wrong

values = [0, 0, 1]  # index into categories
```

**Validation:** `student_values == correct_values` — exact array match.

**MATATAG grades:** 2–10

### C1. Constraint Satisfaction

**`question_mode`:** `"constraint"`

Student enters ANY valid value satisfying multiple simultaneous constraints.
Multiple correct answers accepted.

**Stem:** *"Enter a number that is: greater than 50, a multiple of 7, less
than 100, and odd."*

**Valid answers:** 63, 77, 91

**Skeleton:**
```python
constraints = [
    ("greater than", 50),
    ("multiple of", 7),
    ("less than", 100),
    ("odd", None),
]
# Pre-compute all valid answers for invariant check
valid_set = {x for x in range(1, 1000) if all_pass(x, constraints)}
assert len(valid_set) >= 3  # ensure solvable
```

**Validation:** `all(constraint(student_answer) for constraint in constraints)`

**Cognitive demand:** ANALYSIS — forces genuine search through number space.
Cannot be gamed by elimination.

**MATATAG grades:** 1–10

### C2. Rule Discovery

**`question_mode`:** `"expression"`

A complete (input → output) table is shown. Student types the algebraic rule.
SymPy checks it against all table entries.

**Stem:**
```
| n | output |
|---|---|
| 1 | 3 |
| 2 | 7 |
| 3 | 11 |
| 4 | 15 |
```

**Cognitive demand:** SYNTHESIS — discovering the rule, not applying it.

**MATATAG grades:** 1–8

### C6. Pairs That Work

**`question_mode`:** `"pair_select"`

From a set of 6–8 items, student clicks exactly two that satisfy a stated

### C7. Error Correction (Without Explanation)

**`question_mode`:** `"error_correction"`

Two-step. Step 1: binary judgment (correct/incorrect). Step 2: if incorrect,
enter the correct value. Both validated without AI.

**Stem:** *"A student simplified: $(x + 3)^2 = x^2 + 9$. Is this correct?
If no, enter the correct expansion."*

**Skeleton:**
```python
expression = (x + a)**2
wrong_expansion = x**2 + a**2  # common trap: forgot middle term
correct_expansion = sp.expand(expression)
is_correct = False
```

**Validation:**
- Step 1: `student_judgment == stored_judgment`
- Step 2: `sp.expand(sp.sympify(student_answer)) == correct_expansion`

**Traps used for wrong work generation:**
- `forgot_middle_term`: $(x+a)^2 = x^2 + a^2$
- `wrong_sign`: $(x-a)^2 = x^2 - 2ax - a^2$
- `didnt_square`: $(x+a)^2 = 2x + 2a$

**Cognitive demand:** EVALUATION — identify error + correct it.

**MATATAG grades:** 3–10

### C8. Boundary Finder

**`question_mode`:** `"numeric_input"` (two fields: min, max)

Student enters the minimum AND maximum valid values for an unknown in a
constrained scenario.

**Stem:** *"A rectangle has perimeter 20 cm. Length > width. Both are positive
integers. What are the minimum and maximum possible widths?"*

**Skeleton:**
```python
perimeter = 2 * rng.randint(6, 20)
# l + w = perimeter/2, l > w, both > 0

min_width = 1
max_width = half // 2 - (0 if half % 2 == 0 else 0)
# Exact: w < half/2
```

**Validation:** `student_min == computed_min AND student_max == computed_max`

**Cognitive demand:** ANALYSIS — constraint interpretation, inequality reasoning.

**MATATAG grades:** 4–10

valid. Directly prepares for Gr. 9 formal proof writing.

**MATATAG grades:** 6–10

## Skeleton Schema Extension

The `get_question_skeleton()` return dict is extended with visual fields:

```python
{
    "skeleton_id":     str,
    "variables":       dict,
    "stem_template":   str,
    "correct_answer":  str,
    "options":         dict,         # key → (value_str, trap_name | None)
    "math_expression": str,
    "sympy_correct":   Any,
    # --- Visual extension fields ---
    "visual_type":     str | None,   # e.g., "BarChart", "NumberLine", "BalanceScale"
    "visual_params":   dict | None,  # passed directly to the React component
    "question_mode":   str,          # "mcq" | "numeric_input" | "plotter_bar" | etc.
}
```

---

## Frontend Rendering

The `QuestionResponse` schema includes `visual_type` and `visual_params`. The
React renderer maps `visual_type` to the corresponding component:

```tsx
const VISUAL_COMPONENTS: Record<string, React.FC<any>> = {
  AnalogClock,
  FractionBar,
  FractionCircle,
  NumberLine,
  RightTriangle,
  Polygon,
  SymmetryGrid,
  PatternStrip,
  CoordinatePlane,
  BarChart,
  LineGraph,
  PieChart,
  BoxPlot,
  VennDiagram,
  BalanceScale,
  CurrencyTray,
  GridArea,
  CalendarGrid,
};

// In QuestionCard.tsx:
{visual_type && visual_params && (
  <div className="visual-container">
    {React.createElement(VISUAL_COMPONENTS[visual_type], visual_params)}
  </div>
)}
```

## Interest Theming

`BarChart`, `LineGraph`, `PieChart`, and `PatternStrip` data labels are replaced
by the Gemini narrator using the student's interest theme:

```
default labels:  ["Category A", "Category B", "Category C"]
themed labels:   ["Dinosaurs", "Sharks", "Eagles"]  (interest: animals)
```

The numeric `values` array is NEVER modified by the narrator. Only text labels
and narrative context change. The math is always language-agnostic and
interest-agnostic.

---

## Language Support

`stem_template` is passed to the Gemini narrator with a `language` flag
(`"english"` or `"tagalog"`). Mathematical notation (LaTeX, coordinate pairs,
fractions, equations) is never translated. Only the narrative wrapper (story
context, question phrasing, category labels where text-based) is localized.

## Types Excluded from This Document

These require AI grading and cannot achieve 100% automated validation:

| Type | Reason |
|---|---|
| Free geometric construction | No computable single correct answer |
| Visual error analysis (explain why) | Open-ended explanation |
| Describe-what-you-see open response | Requires language judgment |
| Proof writing with diagram | Logical argument validation |
| "Explain your reasoning" | Natural language output |

### Skipped Visual Types

| Type | Reason |
|---|---|
| EstimationGate | Estimation problems don't benefit from a specialized visual interface. A simple numeric input with tolerance-based validation is more appropriate. The visual "gate" metaphor (showing a range) would reveal too much information to students about the acceptable answer range, defeating the purpose of estimation practice. These competencies are better served by traditional `numeric_input` question mode with backend tolerance checking. |

These are documented in `problem_types.md` under the "skip for now" category.
